What is GitHub?

GitHub is a platform for version control and collaboration. It allows you to host and review code, manage projects, and build software alongside millions of other developers.

Creating a Repository

A repository (repo) is like a folder for your project. To create one, sign in to GitHub, click the "+" icon, and select "New repository". Give it a name, description, and choose if it's public or private.

Cloning a Repository

Cloning means downloading a copy of the repository to your local machine. Use the command: git clone [repository URL] in your terminal.

Making Changes and Committing

After editing files, stage your changes with git add ., then commit them with git commit -m "Your message here". This saves a snapshot of your changes.

Pushing Changes

To upload your commits to GitHub, use git push origin main (assuming your main branch is called 'main').

Pulling Changes

To get the latest changes from the repository, use git pull origin main. This fetches and merges changes from the remote repository.

Branching

Branches allow you to work on different features without affecting the main code. Create a new branch with git checkout -b new-feature, and switch back with git checkout main.

Basic Workflow

A typical workflow: Clone repo → Create branch → Make changes → Commit → Push → Create Pull Request (if collaborating).

Remember, practice makes perfect! Start with small projects to get comfortable with these commands.
